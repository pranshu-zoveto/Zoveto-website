# Marketing copy style (Zoveto)

Use this before shipping new UI copy on the public site.

## Voice

Sound like an operator who has stood inside the workflow:

- Specific about records, handoffs, owners, status, stock, cash, dispatch, payroll, and ledger movement.
- Calm and direct, never motivational.
- Confident without SaaS-template cadence.
- Practical for an owner-founder or ops head reading under time pressure.

## Prefer

- “Operating record” for the shared data layer.
- “Posted event” when a workflow creates a reliable record.
- “Exception queue” when work needs attention.
- “Owner, status, timestamp, next action” for accountability.
- “Stock, cash, orders, people” when summarizing scope.

## Avoid

- “One source of truth,” “built for businesses that mean business,” “seamless,” “unlock,” “supercharge,” “revolutionize.”
- “AI-powered platform,” “inter-process synergy.”
- “Framework” unless legal/compliance text.
- “Real-time” unless the screen proves it.
- “Zero sync errors” as a slogan.
- Generic “scalable / robust / cutting-edge.”

## Rewrite samples

| Weak | Stronger |
| --- | --- |
| One source of truth across teams | One posted operating record for stock, cash, orders, and people |
| Built for businesses that mean business | For teams that cannot close the day on guesswork |
| Real-time operations pulse | Current stock, cash, and order state |
| Inter-process synergy | Handoffs that keep their source record |
| Production-ready architecture | Founder-led implementation reviews |
| AI-powered platform | Rules and AI agents that read posted operations first |

## Ship check

1. Does this sound like someone who has run the workflow?
2. Could this line appear on a thousand SaaS sites?
3. Does it name a real object: stock, order, ledger, user, timestamp, exception, payroll, cash?

If not, rewrite.
